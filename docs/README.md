### Historias de Usuario

La carpeta `historias/` contiene las definiciones detalladas de cada historia de usuario:

- Cada historia incluye:
  - Descripción (Como/Quiero/Para)
  - Criterios de aceptación
  - **Lista detallada de tareas con checkboxes** para seguimiento
  - Notas técnicas
  - Estimación de tiempo
  - Prioridad

- Ver el [README de historias](historias/README.md) para más detalles

## Uso con GitHub Projects

Las historias están diseñadas para facilitar la creación de Issues y el seguimiento en GitHub Projects:

1. **Crear Issues**: Cada historia puede convertirse en un Issue
2. **Subtareas**: Las tareas listadas pueden ser checklists en los Issues
3. **Seguimiento**: Marcar tareas completadas usando los checkboxes
4. **Asignación**: Cada miembro de la pareja trabaja en historias diferentes
